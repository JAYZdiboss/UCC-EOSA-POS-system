-Purpose of program:
This program was created as a functional Point-Of-Sale (POS) interface capable of accepting user input from the start to the end of a shopping interaction. It is capable of processing multiple customers while keeping an updated stock of available store items as well as generating a correct receipt for each customer.

-How to run it:
The user should navigate through the menu options using inputs in the form of numbers ranging from 1-6. Any input outside of this range will lead to an "Invalid option" message, ensuring that it is utilized as instructed. 
The program allows users to display the store's listing of items(Option 1), Add items from that listing to their "Cart"(Option 2), Remove already selected items from their "Cart"(Option 3), display items already in their "Cart"(Option 4) and checkout items(Option 5). 

The user can loop through these menu options until all selected items are finalized in the "Checkout" option, once "Checkout" is selected the user is given one final chance to confirm whether or not they would like to "Checkout" their "Cart" and once confirmation is given with the prompt "Y/y" the program will display the items being checkout out, their unit price, quantity, total for those items as well as the gross total and net total after sales tax is added. If the total bill exceeds $5,000 dollars then a %5 discount is applied to the gross total and is subtracted from the net total. 

	Afterwhich, the user must enter the amount they are paying as an integer and if the amount is less than the net total due; the program will display an "insufficient amount" message and return you to the menu, the cart items will remain and the user would have to intitiate the checkout process once more.
Once the checkout process is completed and the sufficient amount is entered, the program will display a receipt with the company's name, the items purchased, thr subtotal, discount amount(if applicable), sales tax amount, net total due, amount paid and the change as well as a thank you message.

	Finally, a message requesting whether or not you would like to serve another customer will be displayed, if the user input is "Y/y" the cart will be cleared which in a sense allows the program to loop and process multiple customer transactions while updating the stock of items remaining after checkout.
Additionally the program displays a "Low stock warning" whenever you view the store listing and an item falls below "5".

-Assumptions and limitations:
It is assumed that whoever is utilizing this interface is fully capable of reading and following the necessary inputs needed to complete the full shopping and checkout process, in the event the wrong input is entered, a suitable error message should be displayed, returning the user to the menu and allowing them to reattempt the menu option. There are some limitations that will lead to the program crashing all together though such as entering string characters in sections that are expecting an integer or float, for example: the check out option where it requests "The payment amount"
